## 3. Bugs found (severity-rated, with fixes)

### CRITICAL

**C1 — `start_call` crashes on every call (UnboundLocalError)**
- **Location:** `backend/modules/ai/service.py:172` (caused by `:194`)
- **Evidence:** 3 failing API tests; reproduced directly.
- **Cause:** A redundant function-local `from config.settings import settings` at line **194** (inside `if playbook_runtime:`) makes `settings` a *local* variable for the whole `start_call` function. Line **172** (`persona or settings.AI_DEFAULT_PERSONA`) executes first → `UnboundLocalError`. A module-level import already exists at line **54**.
- **Impact:** Breaks **every** call start — live phone calls (`orchestrator.py:303 → _start_call_with_recovery → 423`) AND `/ai/converse`. `UnboundLocalError` is not in the retry filter (`retry_on=(AIMemoryError, AIProviderError)`) and is not an `AIError`, so it propagates and the agent dies before connecting. No opener, no transcript, no conversation. **This single bug fails requirements 1, 2, 5, and 6 at runtime.**
- **Fix:** Delete the redundant local import at line 194 (rely on the module-level `settings` from line 54).

---

### HIGH

**H1 — Call does not self-terminate on idle timeout (runner hang)**
- **Location:** `modules/ai/orchestrator.py:601-607` + `modules/telephony/agent_runner.py:164-168`
- **Cause:** On idle timeout `_event_loop` just `return`s; it never sets `self._stop`. The runner body blocks on `await self._orch._stop.wait()`, and `run()`'s `await loop_task` only runs *after* that body exits. `_stop` is only set by external `stop()` (`orchestrator.py:273`).
- **Impact:** A connected-but-idle call won't hang up itself; finalize/summary is deferred until the Twilio terminal webhook fires `registry.stop()`. "Call ends correctly" on idle fails.
- **Fix:** Call `self.stop()` (or `self._stop.set()`) when the idle timeout fires.

**H2 — STT loop crash also leaves the runner hung**
- **Location:** `orchestrator.py:610-618`
- **Cause:** Same as H1 — the loop-level `except` logs `loop_failed` but never sets `_stop`.
- **Fix:** Set `_stop` in the loop's terminal `except`/`finally`.

**H3 — Publish uses stale config (publish-without-save)**
- **Location:** `frontend/src/pages/Playbooks.tsx:220-234`
- **Cause:** `handlePublish` calls `publishPlaybook(id)` only; unsaved edits to `agent_name`, company fields, and voice are not persisted first.
- **Impact:** Calls run with stale playbook config after a user edits then publishes without an explicit save.
- **Fix:** Save current detail before publish, or block publish while the form is dirty.

**H4 — Playbook-list load errors not surfaced in the dialer**
- **Location:** `frontend/src/hooks/useActivePlaybooks.ts:24-27`, `frontend/src/pages/Calls.tsx:74-78`
- **Cause:** Hook sets `error` but `Calls.tsx` never reads/displays it.
- **Impact:** If `GET /playbooks?active_only=true` fails, the dropdown is silently empty with no message — user cannot tell why they can't place a call.
- **Fix:** Destructure and surface `error` (toast + inline banner).

**H5 — No "missing voice" validation (req 9 gap)**
- **Location:** `Calls.tsx:808-849`, `Playbooks.tsx:165-217`
- **Cause:** No check that the selected playbook has a `voice_id` before Place Call / Publish; backend silently falls back to `ELEVENLABS_VOICE_ID`.
- **Impact:** QA requirement 9 ("missing voice → user-friendly error") is not met; calls proceed on a default voice without warning.
- **Fix:** Add `validateVoice(detail)`; warn/block when `!voice_id`.

## 4. TTS truncation investigation (requirement 8)

**Required log tokens — all present** (with an `ai.orchestrator.` prefix):

| Token | Location |
|-------|----------|
| `AUDIO_PLAYBACK_STARTED` | `orchestrator.py:943-948` |
| `AUDIO_PLAYBACK_COMPLETED` | `orchestrator.py:980-986` |
| `AUDIO_INTERRUPTED` | `orchestrator.py:996-1004` |
| `BARGE_IN_TRIGGERED` | `orchestrator.py:721-727` |

**Is TTS still being interrupted?**
- **On phone, by default: NO via barge-in.** `PHONE_CALL_BARGE_IN_ENABLED=False` suppresses VAD/partial barge-in while the agent is speaking (`orchestrator.py:624-651`, `settings.py:101`). The opener and replies should play to completion. This is the intended fix for the earlier self-echo truncation, and `ignore_identities` correctly stops the agent transcribing its own published track (`orchestrator.py:216-226, 332`; `transport.py:285-293`).
- **Residual truncation risk remains** (Medium): (a) a caller **FINAL** transcript preempts the in-progress utterance even with barge-in suppressed (M2); (b) PSTN **acoustic echo** can generate that FINAL while the agent is speaking — `ignore_identities` does not block the caller's SIP track echoing the agent; (c) `AUDIO_PLAYBACK_COMPLETED` is logged before playout actually finishes (M1), so logs may *understate* truncation.

**Conclusion:** Self-barge-in truncation is fixed for the default phone path, but full-sentence completion is not guaranteed under echo/early-FINAL conditions, and the completion telemetry is misleading. Recommend M1 + M2 before declaring TTS truncation resolved.
